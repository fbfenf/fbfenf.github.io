# 🔰 Jekyll(지킬) 초보자 가이드

안녕하세요! **Jekyll**을 처음 접하시는 분들을 위한 아주 쉬운 가이드입니다.
복잡한 프로그래밍 지식 없이도, 이 가이드만 따라 하면 내 웹사이트를 쉽게 관리할 수 있습니다.

---

## 1. Jekyll이 뭔가요? 🤔

**Jekyll**은 **"글만 쓰면 알아서 웹사이트를 만들어주는 요리사"**라고 생각하면 됩니다.

*   **재료**: 여러분이 쓰는 글 (마크다운 파일, `.md`)
*   **요리사**: **Jekyll** (지킬)
*   **완성된 요리**: 예쁜 웹사이트 (HTML 파일)

여러분이 메모장(VS Code 등)에 글을 써서 저장(`push`)하기만 하면, Jekyll 요리사가 알아서 뚝딱뚝딱 웹사이트로 바꿔서 보여줍니다.

---

## 2. 새 글 쓰기 (페이지 만들기) 📝

웹사이트에 새로운 페이지를 추가하고 싶으신가요? 아주 간단합니다.

### 1단계: 파일 만들기
프로젝트 폴더 안에 새로운 파일을 만듭니다. 이름은 영어로 짓고, 뒤에 `.md`를 붙여주세요.
*   예시: `guide.md`, `about.md`, `blog-post.md`

### 2단계: '머리말(Front Matter)' 작성하기
파일의 **맨 윗부분**에 아래 내용을 똑같이 복사해서 붙여넣으세요. 이것은 Jekyll에게 "이 파일의 정보는 이래!"라고 알려주는 명찰입니다.

```yaml
---
layout: default
title: 내 새로운 페이지 제목
permalink: /new-page/
---
```

*   `layout`: `default` (기본 디자인을 쓰겠다는 뜻)
*   `title`: 화면에 보여질 진짜 제목
*   `permalink`: 이 페이지의 주소 (예: `fbfenf.github.io/new-page/`)

### 3단계: 본문 쓰기
머리말 아래에 자유롭게 내용을 적으세요. (`# 제목`, `- 목록` 등 마크다운 문법 사용 가능)

---

## 3. 사이트 설정 바꾸기 (`_config.yml`) ⚙️

`_config.yml` 파일은 사이트 전체의 **제어 센터**입니다.
이 파일을 열어서 제목이나 설명을 바꿀 수 있습니다.

```yaml
title: 내 사이트 이름
description: 사이트 설명
theme: minima  # 디자인 테마 (건드리지 마세요!)
```

**주의**: 이 파일의 내용을 바꿀 때는 형식을 정확히 지켜야 합니다. (`항목: 값` 형태, 콜론 뒤에 띄어쓰기 필수!)

---

## 4. 사이트 옷 갈아입히기 (테마 변경) 🎨

배경, 글씨체 같은 디자인을 바꾸고 싶으신가요? `theme` 설정만 바꾸면 됩니다.

### 인기 있는 기본 테마
GitHub Pages는 여러 가지 예쁜 테마를 무료로 제공합니다.

| 테마 이름 | 특징 |
| --- | --- |
| **minima** | 가장 기본적이고 깔끔한 블로그 스타일 (현재 적용됨) |
| **cayman** | 파란색 배경의 모던하고 시원한 비즈니스 스타일 |
| **slate** | 개발자 느낌이 나는 어두운(Dark) 스타일 |
| **merlot** | 고급스러운 붉은색 포인트가 있는 스타일 |

### 테마 바꾸는 법
1.  `_config.yml` 파일을 엽니다.
2.  `theme` 부분을 찾아서 원하는 이름으로 바꿉니다.

```yaml
theme: cayman  # minima를 지우고 cayman으로 변경!
```

3.  저장하고(`Commit`) 배포(`Push`)하면 끝! (1~3분 뒤 바뀝니다)

---

## 5. 저장하고 배포하기 🚀

작성을 마쳤다면, GitHub에 올려서 세상에 공개합시다.

1.  **저장 (Commit)**: "새 페이지 추가했음" 하고 기록 남기기
2.  **보내기 (Push)**: GitHub 저장소로 파일 보내기

GitHub에 파일이 올라가면, **약 1~3분 뒤**에 여러분의 주소(`https://아이디.github.io/페이지주소/`)에서 확인 가능합니다.
