## Antigravity란 무엇인가요?

**Antigravity**는 자연어로 대화하며 코드를 작성, 수정, 디버깅할 수 있는 AI 페어 프로그래머입니다.

### 🎯 주요 특징

- **강력한 AI**: Google의 최신 AI 모델 기반
- **자율 작업**: 복잡한 작업을 단계별로 스스로 수행
- **멀티모달**: 코드, 터미널, 브라우저, 이미지 생성 모두 가능
- **컨텍스트 이해**: 프로젝트 전체를 이해하고 작업
- **한국어 지원**: 자연스러운 한국어 대화

### 💡 Antigravity로 할 수 있는 것

```markdown
✅ 새 프로젝트 생성 (웹앱, 모바일 앱, API 등)
✅ 기존 코드 분석 및 리팩토링
✅ 버그 수정 및 디버깅
✅ 문서 작성 (README, 가이드 등)
✅ 테스트 코드 작성
✅ 웹사이트 디자인 및 구현
✅ 데이터 분석 및 시각화
✅ 터미널 명령어 자동 실행
✅ UI/UX 이미지 생성
```

## 핵심 기능

### 1️⃣ 스마트 코드 편집

Antigravity는 파일의 특정 부분만 정확하게 수정합니다.

**예시:**
```
"index.js의 handleSubmit 함수에 유효성 검사를 추가해줘"
```

### 2️⃣ 자율 작업 모드

복잡한 작업을 여러 단계로 나누어 자동 수행합니다.

**작업 흐름:**
1. **Planning (계획)**: 작업을 분석하고 계획 수립
2. **Execution (실행)**: 코드 작성 및 수정
3. **Verification (검증)**: 테스트 및 확인

### 3️⃣ 파일 시스템 탐색

프로젝트 구조를 이해하고 필요한 파일을 찾습니다.

```bash
# 파일 검색
"프로젝트에서 API 관련 파일을 찾아줘"

# 디렉토리 구조 확인
"src 폴더의 구조를 보여줘"

# 코드 검색
"useState를 사용하는 모든 파일을 찾아줘"
```

### 4️⃣ 터미널 통합

명령어를 제안하고 실행합니다.

```bash
"npm 패키지를 설치해줘"
"개발 서버를 실행해줘"
"Git 커밋하고 푸시해줘"
```

### 5️⃣ 브라우저 자동화

웹사이트 테스트 및 정보 수집이 가능합니다.

```
"내 웹사이트를 열고 로그인 기능을 테스트해줘"
"React 공식 문서에서 Hooks 사용법을 찾아줘"
```

### 6️⃣ 이미지 생성

UI 디자인, 로고, 아이콘 등을 생성합니다.

```
"로그인 페이지 디자인을 만들어줘"
"회사 로고를 만들어줘 - 파란색, 미니멀한 스타일"
```

## 시작하기

### 🎬 첫 대화 시작하기

Antigravity와 대화하는 방법은 간단합니다. 자연스럽게 요청하면 됩니다!

#### ✅ 좋은 요청 예시

```markdown
"React로 To-Do 앱을 만들어줘"
"이 함수에 주석을 추가해줘"
"package.json에 express를 추가해줘"
"현재 프로젝트의 구조를 설명해줘"
"버그를 찾아서 수정해줘"
```

#### ❌ 피해야 할 요청

```markdown
"코드 짜줘" (너무 모호함)
"이거 좀" (무엇을 원하는지 불명확)
"전부 다 고쳐줘" (범위가 너무 넓음)
```

### 💬 효과적인 대화법

1. **구체적으로 요청하기**
   ```
   ❌ "회원가입 기능 만들어줘"
   ✅ "React로 회원가입 폼을 만들어줘. 이메일, 비밀번호 필드와 유효성 검사 포함"
   ```

2. **맥락 제공하기**
   ```
   "이 프로젝트는 Next.js 13 App Router를 사용하고 있어. 
   /api/users 엔드포인트를 추가해줘"
   ```

3. **단계별로 나누기**
   ```
   작업이 크면 단계를 나눠서 요청:
   1단계: "먼저 데이터베이스 스키마를 설계해줘"
   2단계: "이제 API 엔드포인트를 만들어줘"
   3단계: "프론트엔드 컴포넌트를 만들어줘"
   ```

## 터미널 명령어

### 🖥️ 명령어 실행

Antigravity가 명령어를 제안하고 승인 후 자동 실행합니다.

**패키지 관리:**
```bash
"React를 설치해줘"
→ npm install react

"종속성을 업데이트해줘"
→ npm update
```

**서버 실행:**
```bash
"개발 서버를 시작해줘"
→ npm run dev

"프로덕션 빌드를 만들어줘"
→ npm run build
```

**Git 작업:**
```bash
"변경사항을 커밋해줘"
→ git add .
→ git commit -m "Update code"

"GitHub에 푸시해줘"
→ git push origin main
```

### ⚙️ 자동 실행 vs 수동 승인

- **안전한 명령어**: 자동 실행 (예: `ls`, `cat`)
- **위험한 명령어**: 승인 필요 (예: `rm`, `npm install`)

## 프로젝트 관리

### 📋 작업 계획

복잡한 프로젝트는 Antigravity가 자동으로 계획을 세웁니다.

**Planning Mode (계획 단계):**
- 요구사항 분석
- 파일 구조 설계
- 작업 단계 정의
- Implementation Plan 문서 생성

**Implementation Plan 예시:**
```markdown
# 블로그 시스템 구현

## 작업 단계
1. 데이터베이스 스키마 설계
2. API 엔드포인트 구현
3. 프론트엔드 컴포넌트 작성
4. 스타일링 및 반응형 디자인
5. 테스트 작성

## 검증 계획
- 단위 테스트 실행
- 통합 테스트
- 브라우저 테스트
```

### ✅ 작업 추적

`task.md` 파일을 통해 진행 상황을 추적합니다:

```markdown
## 체크리스트
- [x] 데이터베이스 스키마 설계
- [x] API 엔드포인트 구현
- [/] 프론트엔드 컴포넌트 작성 (진행중)
- [ ] 스타일링
- [ ] 테스트 작성
```

### 📖 문서화

작업 완료 후 `walkthrough.md`를 생성합니다:

```markdown
# 작업 완료 리포트

## 변경사항
- 블로그 CRUD API 구현
- React 컴포넌트 작성
- CSS 스타일링 완료

## 테스트 결과
- ✅ 모든 API 엔드포인트 정상 작동
- ✅ 프론트엔드 렌더링 확인
- ✅ 반응형 디자인 적용

## 다음 단계
- 사용자 인증 추가
- 댓글 기능 구현
```

## 고급 기능

### 🧠 컨텍스트 이해

Antigravity는 대화 내역과 프로젝트 전체를 기억합니다.

```
USER: "React 프로젝트를 만들어줘"
ANTIGRAVITY: [프로젝트 생성]

USER: "거기에 라우팅을 추가해줘"
ANTIGRAVITY: [방금 만든 프로젝트에 React Router 추가]

USER: "홈 페이지를 만들어줘"
ANTIGRAVITY: [라우팅 설정을 고려하여 컴포넌트 생성]
```

### 🔄 반복 개선

```
USER: "블로그 포스트 카드를 만들어줘"
ANTIGRAVITY: [기본 디자인 생성]

USER: "그림자를 더 진하게 해줘"
ANTIGRAVITY: [스타일 수정]

USER: "호버 효과를 추가해줘"
ANTIGRAVITY: [애니메이션 추가]
```

### 📚 Knowledge Items (KI)

Antigravity는 중요한 정보를 Knowledge Items로 저장합니다.

**활용 예:**
- 프로젝트 아키텍처 패턴
- 버그 해결 방법
- 설정 가이드
- 디자인 시스템

```
"이 프로젝트의 아키텍처를 KI로 저장해줘"
"나중에 참고할 수 있도록 이 설정을 저장해줘"
```

### 🔗 과거 대화 참조

```
"저번에 만든 로그인 컴포넌트를 수정해줘"
"어제 논의한 API 구조를 구현해줘"
```

## 팁과 트릭

### 💡 효율적인 사용법

1. **명확한 요청**
   ```
   ✅ "Button.jsx에서 클릭 이벤트 핸들러를 추가해줘"
   ❌ "버튼 좀 고쳐줘"
   ```

2. **파일명 명시**
   ```
   ✅ "src/components/Header.jsx의 로고 크기를 변경해줘"
   ❌ "헤더의 로고를 바꿔줘"
   ```

3. **예시 제공**
   ```
   "이런 스타일로 만들어줘:
   - 배경: 그라디언트 (파랑 → 보라)
   - 폰트: 모던하고 깔끔한 스타일
   - 버튼: 둥근 모서리"
   ```

4. **단계별 진행**
   ```
   큰 작업은 나눠서:
   1. "먼저 데이터 모델을 설계해줘"
   2. "이제 API를 만들어줘"
   3. "프론트엔드를 만들어줘"
   ```

**템플릿 활용:**
```
"Express API 서버 보일러플레이트를 만들어줘"
"React + TypeScript + Vite 프로젝트를 생성해줘"
```

**일괄 작업:**
```
"모든 .js 파일을 .ts로 변환해줘"
"프로젝트의 모든 console.log를 제거해줘"
```

**자동 문서화:**
```
"이 함수들에 JSDoc 주석을 추가해줘"
"README를 업데이트해줘"
```

### 🎯 베스트 프랙티스

1. **검토 요청하기**
   ```
   "변경사항을 보여주고 확인받을게요"
   ```

2. **설명 요청하기**
   ```
   "이 코드가 어떻게 작동하는지 설명해줘"
   "왜 이 방법을 선택했는지 알려줘"
   ```

3. **대안 탐색하기**

## 자주 묻는 질문

### ❓ Q1: Antigravity가 내 프로젝트를 어떻게 이해하나요?

**A:** Antigravity는 다음을 통해 프로젝트를 이해합니다:
- 파일 구조 탐색
